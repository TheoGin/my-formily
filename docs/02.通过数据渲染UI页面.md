## 阶段1：项目初始化 & 静态表单

### 🎯 本课目标

- 使用 Vite 创建 React + TypeScript 项目
- 集成 Tailwind CSS v4
- 创建基础的 `Input` 和 `Select` 组件
- 搭建一个**硬编码**的静态表单页面

### 1.1 创建项目

```bash
pnpm create vite
cd myformily
pnpm install
```

安装 Tailwind CSS v4（Vite 插件模式）：

```bash
pnpm add tailwindcss @tailwindcss/vite
```

### 1.2 配置 Tailwind

修改 `vite.config.ts`：

```ts
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
  plugins: [react(), tailwindcss()],
});
```

修改 `src/index.css`，只保留一行：

```css
@import "tailwindcss";
```

### 1.3 创建基础组件

**`src/components/Input.tsx`**

```tsx
// components/Input.tsx
type Props = {
  value: string;
  onChange: (value: string) => void;
};

export function Input({ value, onChange }: Props) {
  return (
    <input
      value={value}
      onChange={(e) => onChange(e.target.value)}
      className="w-full rounded border border-gray-300 px-3 py-2
                 focus:outline-none focus:ring-2 focus:ring-blue-500"
    />
  );
}
```

**`src/components/Select.tsx`**

```tsx
// components/Select.tsx
type Props = {
  value: string;
  options?: string[];
  onChange?: (value: string) => void;
};

export function Select({ value, options, onChange }: Props) {
  return (
    <select
      value={value}
      onChange={(e) => onChange && onChange(e.target.value)}
      className="w-full rounded border border-gray-300 px-3 py-2
                 focus:outline-none focus:ring-2 focus:ring-blue-500"
    >
      <option value="">请选择</option>
      {options?.map((opt) => (
        <option key={opt} value={opt}>
          {opt}
        </option>
      ))}
    </select>
  );
}
```

> **设计思路**：这两个组件都是**受控组件**，对外暴露统一的 `value` + `onChange` 接口。这一点非常关键——后续的表单系统正是依靠这个统一接口来管理所有字段。

### 1.4 搭建静态表单

此时我们还没有任何表单系统，先用 React `useState` 硬编码一个静态表单，确保 UI 和组件能跑通。

**`src/App.tsx`**

```tsx
import { useState } from "react";
import { Input } from "./components/Input";
import { Select } from "./components/Select";

function App() {
  const [username, setUsername] = useState("");
  const [type, setType] = useState("");

  return (
    <form className="max-w-md mx-auto mt-10 rounded-lg border bg-white p-6 shadow">
      <h2 className="mb-6 text-xl font-semibold">用户信息表单</h2>

      <div className="mb-4">
        <label className="block mb-1 text-sm font-medium text-gray-700">
          用户名
        </label>
        <Input value={username} onChange={setUsername} />
      </div>

      <div className="mb-4">
        <label className="block mb-1 text-sm font-medium text-gray-700">
          用户类型
        </label>
        <Select value={type} options={["个人", "企业"]} onChange={setType} />
      </div>

      <button
        type="submit"
        className="rounded bg-blue-500 px-4 py-2 text-white hover:bg-blue-600"
        onClick={(e) => {
          e.preventDefault();
          console.log({ username, type });
        }}
      >
        提交
      </button>
    </form>
  );
}

export default App;
```

### 1.5 运行效果

```bash
pnpm dev
```

你应该能看到一个包含"用户名"输入框和"用户类型"下拉框的简单表单。输入内容后点击提交，控制台会打印出表单数据。

## 阶段2：Schema 驱动渲染

### 🎯 本课目标

- 定义 `FieldSchema` 类型，用数据描述表单结构
- 创建 `componentMap` 组件映射表
- 根据 Schema 数组动态渲染表单字段

### 2.1 为什么需要 Schema 驱动？

Schema 驱动的核心思想是：

> **用数据（JSON）来描述表单结构，由引擎统一渲染。**

这样做的好处：

- ✅ 新增字段只需在 Schema 中加一条配置
- ✅ 表单结构可以从后端动态下发
- ✅ 渲染逻辑集中，容易维护

### 2.2 定义 Schema 类型

**`src/schema.ts`**

```ts
// schema.ts

// 先只定义最基础的字段描述
export type FieldSchema = {
  name: string; // 字段唯一标识
  label: string; // 展示标签
  component: "Input" | "Select"; // 使用哪个组件
  options?: string[]; // Select 专用选项
  initialValue?: any; // 初始值
};

export const schema: FieldSchema[] = [
  {
    name: "username",
    label: "用户名",
    component: "Input",
  },
  {
    name: "type",
    label: "用户类型",
    component: "Select",
    options: ["个人", "企业"],
  },
  {
    name: "companyName",
    label: "企业名称",
    component: "Input",
  },
  {
    name: "taxId",
    label: "税号",
    component: "Input",
  },
  {
    name: "contactType",
    label: "联系方式",
    component: "Select",
    options: ["手机号", "邮箱"],
  },
  {
    name: "contact",
    label: "联系方式",
    component: "Input",
  },
];
```

### 2.3 组件映射表

**`src/componentMap.ts`**

```ts
import { Input } from "./components/Input";
import { Select } from "./components/Select";

export const componentMap = {
  Input,
  Select,
};
```

> Schema 里写的 `component: 'Input'` 是一个字符串，`componentMap` 的作用就是把字符串映射到真正的 React 组件。

### 2.4 动态渲染表单

现在改造 App，根据 Schema 动态渲染：

**`src/App.tsx`**

```tsx
import { useState } from "react";
import { schema } from "./schema";
import { componentMap } from "./componentMap";

function App() {
  // 用一个对象统一管理所有字段的值
  const [values, setValues] = useState<Record<string, any>>(() => {
    const init: Record<string, any> = {};
    schema.forEach((item) => {
      init[item.name] = item.initialValue ?? "";
    });
    return init;
  });

  return (
    <form className="max-w-md mx-auto mt-10 rounded-lg border bg-white p-6 shadow">
      <h2 className="mb-6 text-xl font-semibold">用户信息表单</h2>

      {schema.map((fieldSchema) => {
        const Component = componentMap[fieldSchema.component];

        return (
          <div key={fieldSchema.name} className="mb-4">
            <label className="block mb-1 text-sm font-medium text-gray-700">
              {fieldSchema.label}
            </label>
            <Component
              value={values[fieldSchema.name]}
              options={fieldSchema.options}
              onChange={(val: any) =>
                setValues((prev) => ({ ...prev, [fieldSchema.name]: val }))
              }
            />
          </div>
        );
      })}

      <button
        type="submit"
        className="rounded bg-blue-500 px-4 py-2 text-white hover:bg-blue-600"
        onClick={(e) => {
          e.preventDefault();
          console.log(values);
        }}
      >
        提交
      </button>

      {/* 预览当前表单数据 */}
      <pre className="mt-6 text-left">{JSON.stringify(values, null, 2)}</pre>
    </form>
  );
}

export default App;
```

### 2.5 运行效果

```bash
pnpm dev
```

现在你看到的表单界面和上一课差不多，但背后的渲染方式完全不同——**所有字段都是从 schema 数组动态生成的**。
